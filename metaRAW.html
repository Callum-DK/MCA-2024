How would I improve this?:

Use of field names avoiding hyphens, spaces, slashes, and special characters. I could use Excel and CSV by creating columns, with the field name in the first column and the field value in the second column. I could then link this excel document as a downloadable file or set up a preview. An interesting find was Markdown. I could use Markdown files to store metadata that requires a list, such as tags. I can use Markdown to bolden important text. Markdown files can be converted to HTML to be displayed for viewing. I could make use of exporting by using a drop-down list to export metadata to a standard format, such as FGDC CSDGM, ISO 19115-3, ISO 19139, or INSPIRE. 

I noticed that the order in which the data was presented was chronological in relation to each line of code from the MEI file. I would first like to change the order but rearranging the code so that the data is displayed in a logical order.

Title, Composer, Publisher, Publication Date, Distributor, Genre, License, Encoding Date.

Upon attempting this, I realise that this isn’t as simple as it seems. Due to certain Parent elements containing certain child elements following the MEI Guidelines and metadata standards, this means we cannot place certain metadata into the wrong containers.

I realised that I wanted to add a publication date as I thought this would be useful in relation to the license. It will give a user an idea on how old or new the piece is which can justify the permissions granted or not granted via licensing. I believe it will encourage users to stay up to data on an artists work and decide how relevant a piece of work is in relation to the time period the user is in.
